# iOS Voice Keyboard Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a native iOS voice keyboard (Wispr-Flow-style) where you tap a button in any app, speak, and get cleaned-up text inserted at the cursor — reusing the macOS app's transcription + cleanup pipeline.

**Architecture:** An iOS *containing app* holds the microphone and runs the pipeline (iOS keyboard extensions cannot access the mic); a thin *keyboard extension* shows the UI and inserts text; a *Live Activity widget* shows the session on the Dynamic Island. The three processes communicate over an App Group container + Darwin notifications. A "Full Flow Session" (a held background-audio session with an auto-end timer) keeps the app alive so dictation repeats without re-launching the app. Transcription is batch per-utterance via the existing Groq/OpenAI-compatible pipeline, extracted into a multiplatform Swift package shared by macOS and iOS.

**Tech Stack:** Swift, SwiftUI, AVFoundation (`AVAudioSession`/`AVAudioEngine`), ActivityKit (Live Activities), App Intents, `CFNotificationCenter` (Darwin notifications), App Groups, SwiftPM (shared package), XcodeGen (generates the iOS `.xcodeproj` from `ios/project.yml`, keeping the repo's text-based, no-committed-xcodeproj convention), XCTest.

**Spec:** [docs/superpowers/specs/2026-06-09-ios-voice-keyboard-design.md](../specs/2026-06-09-ios-voice-keyboard-design.md)

## Read this first: how to use this plan

- **Phase 0 is a gate.** It is a throwaway spike that proves the two fragile iOS mechanics (launching the app from the keyboard; the keyboard↔backgrounded-app round-trip). Do NOT build Phases 3–9 until the spike passes; its findings (which launch mechanism works, exact ActivityKit/AVAudioSession behavior on the current iOS) finalize the integration code. If the spike fails the smooth path, fall back to the per-dictation app-bounce (still in-scope) — the engine and keyboard tasks are written so that fallback is a small change.
- **Phases 1–2 are pure logic** (a Swift package + Codable/state types). They are full TDD with `swift test` and can be done in parallel with / before the spike.
- **Phases 3–9 are iOS system integration.** Much of this is not unit-testable; "verify" steps are build-run-observe on a **real device** (keyboard extensions, background audio, mic, and Live Activities do not behave correctly in older simulators; use a device). Where a step writes logic that *is* testable (timers, validity windows, heartbeat math), it is split into a testable helper with its own test.
- **Confirm fast-moving APIs during the spike.** ActivityKit background-start rules and the keyboard→app launch are the parts most likely to have shifted; the spike is where you pin them down against current Apple docs (use the context7 MCP or developer.apple.com).
- **Prerequisite (human):** an Apple Developer Program membership and a registered App Group + bundle ids are required before Phase 0 can run on a device. See Phase 9 for the full distribution checklist.

**Conventions used below**
- App Group id: `group.com.vishk23.whisprfreeme` (override everywhere if you pick another).
- Bundle ids: app `com.vishk23.whisprfreeme.ios`, keyboard `com.vishk23.whisprfreeme.ios.keyboard`, widget `com.vishk23.whisprfreeme.ios.widget`.
- Darwin notification names: `com.vishk23.whisprfreeme.dictation.start`, `.stop`, `.done`, `.heartbeat`, `.heartbeatAck`, `.sessionStart`, `.sessionEnd`.
- iOS deployment target: **17.0** (interactive Live Activities / App Intents).

```swift
import Foundation

public enum SessionTimeout: String, Codable, Sendable, CaseIterable {
    case minutes5, minutes15, hour1, never
    public var seconds: TimeInterval? {
        switch self { case .minutes5: 300; case .minutes15: 900; case .hour1: 3600; case .never: nil }
    }
}

public struct SyncableSettings: Codable, Equatable, Sendable {
    public var schemaVersion: Int
    public var customVocabulary: [String]
    public var customSystemPrompt: String?
    public var sessionTimeout: SessionTimeout
    public var hapticsEnabled: Bool

    enum CodingKeys: String, CodingKey {
        case schemaVersion
        case customVocabulary = "custom_vocabulary"
        case customSystemPrompt = "custom_system_prompt"
        case sessionTimeout
        case hapticsEnabled
    }
    public init(from decoder: Decoder) throws {
        let c = try decoder.container(keyedBy: CodingKeys.self)
        schemaVersion = (try? c.decode(Int.self, forKey: .schemaVersion)) ?? 1
        customVocabulary = (try? c.decode([String].self, forKey: .customVocabulary)) ?? []
        customSystemPrompt = try? c.decodeIfPresent(String.self, forKey: .customSystemPrompt)
        sessionTimeout = (try? c.decode(SessionTimeout.self, forKey: .sessionTimeout)) ?? .minutes15
        hapticsEnabled = (try? c.decode(Bool.self, forKey: .hapticsEnabled)) ?? true
    }
    public init(schemaVersion: Int = 1, customVocabulary: [String] = [], customSystemPrompt: String? = nil, sessionTimeout: SessionTimeout = .minutes15, hapticsEnabled: Bool = true) {
        self.schemaVersion = schemaVersion; self.customVocabulary = customVocabulary
        self.customSystemPrompt = customSystemPrompt; self.sessionTimeout = sessionTimeout; self.hapticsEnabled = hapticsEnabled
    }
    public static let `default` = SyncableSettings()
}
```
